# 对话系统与前端重构设计文档

## 概述
为会议纪要智能体添加 AI 对话能力，并重构前端为 ChatGPT 风格布局。

## 核心变更

### 1. AI 对话集成
- **LLM**: DeepSeek V4 Flash（云端API，兼容OpenAI格式）
- **接入方式**: `POST https://api.deepseek.com/v1/chat/completions` + `stream: true`
- **传输方式**: 后端 SSE (SseEmitter) 流式推送到前端
- **上下文**: 基于对话历史，**不做 RAG**（不注入会议转写内容）
- **配置**: API Key 通过 `DEEPSEEK_API_KEY` 环境变量注入

### 2. 交互流程
1. 用户在聊天框输入消息 → 前端 POST 到后端
2. 后端将用户消息存入数据库
3. 后端取出完整对话历史，拼成 DeepSeek 请求
4. 后端调用 DeepSeek 流式 API，逐 token 通过 SSE 推送给前端
5. 前端实时渲染打字机效果
6. 流结束后，完整 AI 回复存入数据库

### 3. 前端布局重构
从当前卡片堆叠布局改为 ChatGPT 风格左右布局：

```
┌──────────────────────────────────────────────┐
│  Header: 会议纪要智能体 (深色)                │
├────────────┬─────────────────────────────────┤
│  Sidebar   │  Chat Area                      │
│  (280px)   │                                 │
│ [新建对话]  │  无选中时 → 欢迎语              │
│            │  有对话时 → 消息气泡列表 + 输入框 │
│ 💬 对话列表 │  用户气泡右对齐(品牌色)           │
│            │  AI 气泡左对齐(浅灰)              │
│ ────────   │  流式输出显示打字机效果           │
│ 📋 会议列表 │  支持 Markdown 渲染              │
│ 🔍 搜索    │                                 │
└────────────┴─────────────────────────────────┘
```

### 6. 约束
- AI 回复**不**包含会议转写内容（无 RAG）
- 仅依赖对话历史消息做上下文
- SSE 连接超时 5 分钟
- 使用 Java 21 HttpClient（无需新增依赖）
