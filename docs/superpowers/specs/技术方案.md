┌─────────────────────────────────────────────────────────────┐
│                  AgentScope Java 框架                     │

## 2. Agent 架构设计

### 2.1 Agent 职责划分

#### MeetingAgent（主控制器）
- **职责**：协调整个处理流程，管理 Agent 间协作
- **功能**：
  - 接收用户请求
  - 协调子 Agent 工作
  - 返回处理结果
  - 错误处理和重试

#### FileProcessingAgent（文件处理）
- **职责**：处理视频和音频文件
- **功能**：
  - 本地文件读取
  - 音频提取（FFmpeg）
  - 文件格式验证
  - 临时文件管理

#### TranscriptionAgent（语音转写）
- **职责**：将音频转换为文字
- **功能**：
  - 调用 FunASR 模型
  - 语音识别处理
  - 结果格式化
  - 质量检查

#### SearchAgent（搜索检索）
- **职责**：实现 RAG 检索功能
- **功能**：
  - 向量化处理
  - 全文检索
  - 语义搜索
  - 结果融合排序

#### RewriteAgent（风格改写）
- **职责**：按指定风格改写内容
- **功能**：
  - 风格分析
  - 内容改写
  - 风格迁移
  - 结果优化

### 2.2 Agent 通信机制

- **工具调用**：Agent 间通过 @Tool 注解暴露功能
- **共享记忆**：使用 AgentScope Memory 共享状态
- **消息传递**：通过事件机制进行通知
- **异步处理**：支持异步任务执行

## 3. 数据架构设计

### 3.1 数据表结构

#### 会议纪要表 (meeting_minutes)
```sql
CREATE TABLE meeting_minutes (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,          -- 会议主题
    content TEXT NOT NULL,               -- 完整会议纪要内容
    meeting_time TIMESTAMP WITH TIME ZONE, -- 会议时间
    location VARCHAR(255),               -- 会议地点
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 向量表 (meeting_vectors)
```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE meeting_vectors (
    id BIGSERIAL PRIMARY KEY,
    meeting_id BIGINT REFERENCES meeting_minutes(id) ON DELETE CASCADE,
    chunk_number INT,                    -- 第几个chunk
    chunk_content TEXT NOT NULL,        -- chunk内容
    embedding vector(1536),              -- 1536维向量
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 AgentScope Memory 设计

#### 短期记忆
- 当前处理的会话状态
- 临时处理结果
- Agent 间共享数据

#### 长期记忆
- 已存储的会议纪要
- 向量化索引
- 用户偏好设置

## 4. 工作流程设计

### 4.1 第一阶段：前端 + 转写
```
用户上传视频 → FileProcessingAgent 提取音频 → 
TranscriptionAgent 语音转写 → 返回文字结果
```

### 4.2 第二阶段：RAG检索
```
会议数据存储 → SearchAgent 向量化 → 
支持混合搜索（关键词+语义）→ 返回相关结果
```

### 4.3 第三阶段：风格改写
```
RewriteAgent 分析风格特征 → 
根据用户选择进行风格迁移 → 
返回改写后的纪要
```

## 5. Agent 实现设计

### 5.1 Agent 基础结构
```java
@Agent(name = "MeetingAgent")
public class MeetingAgent {
    
    @Autowired
    private FileProcessingAgent fileAgent;
    
    @Autowired
    private TranscriptionAgent transcriptionAgent;
    
    @Autowired
    private SearchAgent searchAgent;
    
    @Autowired
    private RewriteAgent rewriteAgent;
    
    // 工作流编排
    public String processMeeting(String videoPath) {
        // 1. 文件处理
        String audioPath = fileAgent.processVideo(videoPath);
        
        // 2. 语音转写
        Transcript transcript = transcriptionAgent.transcribe(audioPath);
        
        // 3. 存储到记忆
        String meetingId = memory.saveMeeting(transcript);
        
        // 4. 向量化处理
        searchAgent.vectorizeMeeting(meetingId, transcript);
        
        return meetingId;
    }
}
```

### 5.2 工具暴露机制
```java
@Agent(name = "FileProcessingAgent")
public class FileProcessingAgent {
    
    @Tool(name = "extractAudio", description = "从视频中提取音频")
    public String extractAudio(String videoPath) {
        // 使用 FFmpeg 处理
        return ffmpegService.extractAudio(videoPath);
    }
}
```

### 5.3 异步处理支持
```java
@Async
public void vectorizeMeeting(String meetingId, Transcript transcript) {
    // 异步向量化处理
    List<TextChunk> chunks = chunkContent(transcript.getText());
    for (TextChunk chunk : chunks) {
        Vector embedding = generateEmbedding(chunk.getText());
        vectorStore.add(meetingId, chunk, embedding);
    }
}
```
