### 阶段二：核心功能开发（旧有功能）

#### 8. ~~RAG 向量搜索~~ ⚠️ 已重构为独立 RAG 管线

旧有的 RAG 实现（`VectorizationService` + `SearchService` 基于 `meeting_vectors` 表）已冻结，
新的 RAG 系统基于[独立设计文档](docs/superpowers/rag/rag-design.md)重新实现，使用新的表结构和算法。

**架构变更（2026-06-22）：**

| 维度 | 旧实现 | 新设计 |
|---|---|---|
| 表结构 | `meeting_minutes` + `meeting_vectors` | `document` + `document_chunk`（新表） |
| 会话 | `dialogues` + `dialogue_messages` | `conversation` + `message`（新表） |
| ORM | Spring Data JPA（保留） | 保留 JPA，不引入 MyBatis-Plus |
| Agent | AgentScope HarnessAgent（保留） | 保留 HarnessAgent，预检索注入 |
| 检索算法 | 简单合并去重 | RRF + 时间衰减 + MMR + 证据评估（工具类） |
| 配置中心 | Nacos（保留） | Nacos 管理公共配置，RAG 参数本地 yml |
| 语音识别 | FunASR（保留，独立管线） | 与 RAG 并行，互不依赖 |

**新 RAG 管线表（追加到 init-db.sql）：**
- `document` — RAG 文档/会议转写数据
- `document_chunk` — 文档分块 + 向量（1536 维）
- `conversation` — 新会话表
- `message` — 新消息表

> 旧表（meeting_minutes / meeting_vectors / dialogues / dialogue_messages）继续用于 FunASR 语音转写管线，
> 新表（document / document_chunk / conversation / message）用于 RAG 管线，两条管线通过 ChatService 统一呈现。
